# 17TeamB_blade


![201703233469468747470733a2f2f646e2d6269657a68692e71626f782e6d652f4c4f474f5f4249472e706e67.png](http://oiqdakvix.bkt.clouddn.com/201703233469468747470733a2f2f646e2d6269657a68692e71626f782e6d652f4c4f474f5f4249472e706e67.png?imageView/2/w/1200/q/100)

Blade是一个轻量级的JavaWeb模块，其官方主页可见[BladeJava](https://www.bladejava.com)。

**轻量级**  
新一代JavaWeb框架，不依赖更多的库，摆脱SSH的臃肿，模块化设计，使用起来更轻便！

**简洁高效**  
代码简洁、结构清晰，写更少的代码快速开发！

**方便部署**  
内置Jetty、Tomcat服务器，支持非web方式打成jar包直接运行！

## 文件结构说明


| 路径 | 说明 |
| --- | --- |
| /实验内容 | 存放所有实验的输出产物和成果 |
| /源码 | 存放开发过程的Demo和实例 |
| /过程管理 | 存放会议记录、过程管理的表格等文件 |
| /幻灯片 | 存放每周展示的幻灯片文件 |
| /Project管理.mpp | Microsoft Project文件，记录项目管理内容 |
| /README.md | 项目说明 |

| /过程管理/会议内容笔记 | 存放一些课堂、会议上记录的便签、文档，主要以oneNote为主 |
| /过程管理/文档模板 | 存放文档模板 |
| /过程管理/工作量统计.xls | 记录小组成员日常工作情况 |
| /过程管理/会议记录 | 记录小组成员日常会议的时间、地点、工作任务和会议总结 |
| /过程管理/问题追踪 | 记录小组成员评审、自审期间发现的问题、说明和解决情况 |

# B组实验七/八工具使用手册

我们在实验七（配置管理分析）和实验八（工作量统计分析）中发现了一些重复机械操作的工作，因此使用有效的工具来处理是非常高效的选择。恰好我们发现了一些简单的办法能够一定程度上解决这些问题。以下包括两个工具：

- 基于关键词的Github提交分析工具
- 基于Web的工作量统计工具

## Github提交分析工具

Github的提交列表中，每次只显示30个，对于频繁提交的组来说，统计这些信息是非常耗时的。同时，在Github中充斥了Conflict和Merge等等对实验无意义的操作。我们考虑，当我们在分析版本变化的时候，大脑通过分析提交的描述判断更新内容，事实上是一种对关键词的分析。据此，我们考虑两个方面来解决这部分重复工作：

1. 自动化处理关键词：重复人工行为
2. 约束关键词：减少关键词种类避免干扰，添加特殊意义关键词凸显意义

据此，我们用一点时间开发了GitHubInfo，目前是1.5版本。

### 运行

运行环境为`JDK 1.7+`，通过命令行即可启动，运行命令如下所示：

``` shell

```

### 分析内容

GithubInfo通过GithubApi从commit信息中获取有用的数据，程序总共做了如下几件事情：

1. 验证参数
2. 整理关键词
2. 爬取所有提交信息
3. 通过关键词进行归类和统计
4. 通过提交人员进行归类和统计

如有更多需要可以通过微信联系B组添加。

## 工作量统计工具

工作量统计工具是为了避免每周通过回忆记录周报，使用周报记录存在如下几点不利：

1. 统计不准确，很难确定真正的开始时间和结束时间
2. 短时间的工作，例如10分钟的工作，很难真正的被记录
3. 统计工作需要额外时间开销

因此我们考虑AOP的思路，以工作为切点，在工作前记录时间，在之后记录时间，填写简单的信息，即可完成记录。所以我们考虑通过Web搭建一个简单的记录工具。

工具具有如下几个优势：

1. 响应式布局（毕竟就这么点东西）
2. 分项目管理，同一个项目的信息单独记录，同一个小组的任务统一记录
3. 自动统计和计算工时、百分比
4. 简单易用，记录准确

### 使用方法

登陆 `buaa.hhchat.cn:10000` 或者 `soft.hhchat.cn:10000` ，登陆账户。

- 开始按钮：开启一个任务，开始统计时间
- 提交按钮：结束一个任务，停止统计时间
- 取消按钮：关闭一个任务，任务作废
- 查看统计情况：查看本组本周的工作情况、所有的工作情况以及所有人的工作情况

在整个程序的设计过程中，没有考虑如何维护数据的安全性，因此没有开放注册，如果贵组需要使用，发邮件至`hh23485@126.com`，发送包括小组四个人的**用户名**（不是**邮箱**）和**密码**以及所属的**组**，收到回复后即可使用。

虽然没有考虑数据的安全，不过保证了3个按键，人工点击的情况下，怎么玩数据都不会出错。。。

**需要注意的是：** 开始按钮除了通知服务器开始计时以外，不提交任何数据，输入的信息以提交时为准。可以关闭浏览器，重新打开可以继续操作，但是输入的内容不会保留。
